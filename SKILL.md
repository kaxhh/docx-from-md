# DOCX from Markdown

将 Markdown 文档转换为符合 `.docx` 模板格式的 Word 文档。

## 前置条件

本 skill 的 `scripts/` 目录必须包含以下文件（由 `docx-deterministic-generator` skill 生成后放入）：

- `scripts/fix_md_tables.py` — 表格预处理（修复非矩形管道表格）
- `scripts/md_to_json.py` — MD → JSON 转换器（内嵌 JSON schema）
- `scripts/render_docx.py` — JSON → DOCX 渲染器

如果 `md_to_json.py` 或 `render_docx.py` 不存在，提示用户：

> 请先使用 `$docx-deterministic-generator` 分析你的 `.docx` 模板，生成 `md_to_json.py` 和 `render_docx.py`，然后将它们复制到 `skills/docx-from-md/scripts/` 目录。

## 使用方式

告诉 Claude：

> 用 `$docx-from-md` 把这个 MD 转成 Word

或者直接运行流水线脚本：

```bash
python3 skills/docx-from-md/scripts/run_pipeline.py \
  --md 你的文档.md \
  --template 模板.docx \
  --output generated.docx
```

参数说明：

| 参数 | 必填 | 说明 |
|---|---|---|
| `--md` | ✅ | 输入的 Markdown 文档（按模板章节格式编写） |
| `--template` | ✅ | 原始 `.docx` 模板 |
| `--output` | ✅ | 输出 `.docx` 路径 |
| `--keep-json` | ❌ | 保留中间 JSON 文件（默认删除） |

## 流水线步骤

1. 运行 `fix_md_tables.py`：修复非矩形管道表格（截断多余单元格或补齐空单元格）
2. 检查 `scripts/md_to_json.py` 和 `scripts/render_docx.py` 是否存在
3. 运行 `md_to_json.py`：MD → JSON
4. 验证生成的 JSON 是否符合 schema
5. 运行 `render_docx.py`：JSON → DOCX
6. 清理中间文件（除非指定 `--keep-json`）

## 错误处理

- 非矩形表格 → 自动修复，输出警告
- 脚本不存在 → 提示用户先运行 `docx-deterministic-generator`
- MD 格式错误 → 显示 `md_to_json.py` 的错误信息
- JSON 不符合 schema → 显示验证错误
- DOCX 生成失败 → 显示 `render_docx.py` 的错误信息
